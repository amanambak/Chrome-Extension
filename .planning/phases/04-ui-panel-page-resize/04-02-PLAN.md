<task type="auto">
  <name>Task 1: Implement FixedShiftHelper in content.js</name>
  <files>content.js</files>
  <action>
    - Create a `FixedShiftHelper` utility in `content.js`.
    - Implement `shiftElements(offset)`:
      - Iterate through all elements on the page (or a subset of likely fixed elements like headers/navs).
      - Filter for those with `getComputedStyle(el).position === 'fixed'`.
      - Exclude the extension host itself.
      - If `offset > 0` and element is pinned right (e.g. `right: 0`), increase its `right` style by `offset`.
      - If `offset === 0`, restore original `right` style (use `dataset` to store/restore original values).
    - Update `LayoutManager` to call `FixedShiftHelper.shiftElements(350)` when showing and `(0)` when hiding.
  </action>
  <verify>
    <automated>grep "FixedShiftHelper" content.js</automated>
  </verify>
  <done>Fixed elements can be shifted left programmatically.</done>
</task>

<task type="auto">
  <name>Task 2: Implement MutationObserver for dynamic elements</name>
  <files>content.js</files>
  <action>
    - Initialize a `MutationObserver` in `content.js` to watch the `document.body` for added nodes.
    - When new nodes are added and the panel is currently visible, check if they are `position: fixed` and apply the shift.
    - Debounce the observer's callback to prevent performance issues during large DOM updates.
    - Ensure the observer only runs when the panel is visible.
  </action>
  <verify>
    <automated>grep "MutationObserver" content.js</automated>
  </verify>
  <done>Dynamically added fixed elements are correctly shifted.</done>
</task>
