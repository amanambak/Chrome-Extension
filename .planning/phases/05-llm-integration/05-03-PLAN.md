<task type="auto">
  <name>Task 1: Track Utterance IDs & Trigger LLM</name>
  <files>content.js, utterance-card.js</files>
  <action>
    - Update `UtteranceCard` class to accept an `id` in constructor and store it as `this.id`.
    - In `content.js`:
      - Maintain an internal map `aiCards = new Map()` to track `utteranceId -> AiResponseCard` mappings.
      - Generate a unique `utteranceId` (e.g., `Date.now()`) when creating a new `UtteranceCard`.
      - Update `finalizeCurrentCard()`:
        - Before clearing the card, extract its text.
        - Send a `PROCESS_UTTERANCE` message to the background with: `{ text, utteranceId, tabId }`.
  </action>
  <verify>
    <automated>Verify content.js sends PROCESS_UTTERANCE message on card finalization with correct ID.</automated>
  </verify>
  <done>Finalized utterances correctly trigger background LLM processing.</done>
</task>

<task type="auto">
  <name>Task 2: Route Streaming Response Chunks</name>
  <files>content.js</files>
  <action>
    - Add a listener for `AI_RESPONSE_CHUNK` in `content.js`.
    - Retrieve the `utteranceId` from the message.
    - Check if an `AiResponseCard` already exists in `aiCards` for this ID.
    - If not, create a new `AiResponseCard`, add it to the panel's container (immediately after the matching `UtteranceCard`), and store in the map.
    - Implement prefix-detection logic:
      - Search for `[ANSWER]` or `[SUGGESTION]` in the first few chunks.
      - Call `card.setType()` accordingly.
      - Strip the prefix from the text before calling `card.appendChunk()`.
    - Call `panel.scrollToBottom()` for each chunk to keep the streaming visible.
  </action>
  <verify>
    <automated>Verify content.js listener for AI_RESPONSE_CHUNK creates/updates cards by ID.</automated>
  </verify>
  <done>AI responses correctly stream into the UI, matched to their source transcript segment.</done>
</task>
