<task type="auto">
  <name>Task 1: Deepgram WebSocket Integration in Offscreen Document</name>
  <files>offscreen.js</files>
  <action>
    - Import `CONFIG` from `config.js` (using ESM import if possible, or standard script tag in `offscreen.html`).
    - Implement `openDeepgramConnection()`:
      1. Construct WebSocket URL with parameters from `CONFIG.DEEPGRAM_PARAMS` (nova-3, linear16, 16000, interim_results=true).
      2. Open WebSocket with `['token', CONFIG.DEEPGRAM_API_KEY]` sub-protocol.
      3. Listen for `onopen`, `onmessage`, `onerror`, `onclose`.
    - When `workletNode.port.onmessage` fires:
      1. Send the PCM buffer binary data to the WebSocket if it is open.
    - When `onmessage` from Deepgram fires:
      1. Parse JSON transcript.
      2. Send a message to `background.js` with transcript data.
  </action>
  <verify>
    - DevTools Network tab for `offscreen.html` shows a successful WebSocket connection to Deepgram.
    - Binary messages are being sent continuously.
    - Deepgram responses (JSON) are visible in the "Messages" tab of the WebSocket.
  </verify>
  <done>Offscreen document streams audio to Deepgram and receives transcripts.</done>
</task>

<task type="auto">
  <name>Task 2: Implement Transcript Relay and Content Logging</name>
  <files>background.js, content.js</files>
  <action>
    - In `background.js`:
      1. Store `targetTabId` when capture starts.
      2. Listen for messages from `offscreen.js` containing transcripts.
      3. Use `chrome.tabs.sendMessage(targetTabId, transcriptMessage)` to forward data to the captured tab.
    - In `content.js`:
      1. Listen for transcript messages.
      2. Log them to the console: `[AudioAI] Transcript: "text" (interim/final)`.
  </action>
  <verify>

    - Inspect the YouTube tab console.

  </verify>
