Output: Active audio pipeline in the offscreen document that produces PCM data.

<task type="auto">
  <name>Task 1: Setup AudioContext and Tab Capture</name>
  <files>offscreen.js, offscreen.html</files>
  <action>
    - Update `offscreen.js` to listen for `START_CAPTURE` message containing the `streamId`.
    - Implement `setupCapture(streamId)` function:
      1. Create `new AudioContext({ sampleRate: 16000 })`.
      2. Call `navigator.mediaDevices.getUserMedia({ audio: { mandatory: { chromeMediaSource: 'tab', chromeMediaSourceId: streamId } } })`.
      3. Create `audioContext.createMediaStreamSource(stream)`.
      4. Connect `source` to `audioContext.destination` (CRITICAL for passthrough).
    - Handle capture stop and resource cleanup (closing context).
  </action>
  <verify>
    - Capture starts without console errors in `offscreen.html`.
    - Tab audio continues to play (test with a YouTube video).
  </verify>
  <done>Offscreen document captures tab audio and maintains passthrough.</done>
</task>

<task type="auto">
  <name>Task 2: Implement PCM Conversion via AudioWorklet</name>
  <files>capture-worklet.js, offscreen.js</files>
  <action>
    - Create `capture-worklet.js` implementing `AudioWorkletProcessor`:
      1. Receive `Float32Array` in `process()`.
      2. Convert to `Int16Array` (PCM16).
      3. Use `port.postMessage()` to send PCM buffer back to `offscreen.js`.
    - In `offscreen.js`:
      1. Register the worklet module using `audioContext.audioWorklet.addModule('capture-worklet.js')`.
      2. Create `new AudioWorkletNode(audioContext, 'capture-worklet')`.
      3. Connect `source` to the worklet node.
      4. Listen for `workletNode.port.onmessage` and log the buffer size (temporary verification).
  </action>
  <verify>
    - AudioWorklet loads successfully.
    - Console logs in offscreen document show incoming PCM buffers.
  </verify>
  <done>Audio pipeline converts raw audio to PCM16 chunks.</done>
</task>
