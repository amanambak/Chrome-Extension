<task type="auto">
  <name>Task 2: Implement SW Toggle Logic and Offscreen Lifecycle</name>
  <files>background.js</files>
  <action>
    - Implement `TOGGLE_CAPTURE` message handler in `background.js`.
    - Maintain a global `isCapturing` flag.
    - When starting capture:
      1. Get `streamId` using `chrome.tabCapture.getMediaStreamId({ targetTabId: senderTabId || activeTabId })`.
      2. Create offscreen document using `chrome.offscreen.createDocument({ url: 'offscreen.html', reasons: ['USER_MEDIA'], justification: 'Capture tab audio for transcription' })`.
      3. Store state in `chrome.storage.local`.
    - When stopping capture:
      1. Close offscreen document using `chrome.offscreen.closeDocument()`.
      2. Update state.
    - Handle `chrome.runtime.onInstalled` to clear state.
  </action>
  <verify>
    - Clicking toggle in popup creates `offscreen.html` (check `chrome://extensions` -> "Offscreen Document").
    - Clicking again closes it.
    - State persists when popup closes/reopens.
  </verify>
  <done>Service worker manages capture state and offscreen document lifecycle.</done>
</task>
