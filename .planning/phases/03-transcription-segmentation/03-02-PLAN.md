<task type="auto">
  <name>Task 1: Implement UtteranceCard Component</name>
  <files>utterance-card.js</files>
  <action>
    - Create `utterance-card.js` with a `UtteranceCard` class.
    - The class should manage a `div` element with two child spans:
      - `.stable-text`: for `isFinal: true` segments (white text).
      - `.interim-text`: for `isFinal: false` updates (gray text).
    - Provide methods:
      - `updateInterim(text)`: Updates the interim span.
      - `appendFinal(text)`: Appends text to the stable span and clears interim.
      - `finalize()`: Adds a visual "finished" state (e.g., reduces opacity of interim, or just locks the card).
    - Ensure all text is set using `textContent` to prevent XSS (per research security domain).
  </action>
  <verify>
    <automated>ls utterance-card.js && grep -n "textContent" utterance-card.js</automated>
  </verify>

<task type="auto">
  <name>Task 2: Implement Transcript State Machine in content.js</name>
  <files>content.js</files>
  <action>
    - Update `content.js` to manage `currentCard` and `currentUtteranceBuffer`.
    - Handle `TRANSCRIPT_UPDATE` message:
      - If no `currentCard`, call `panel.addCard(new UtteranceCard())`.
      - If `isFinal === false`: call `currentCard.updateInterim(transcript)`.
      - If `isFinal === true`:
        - Append `transcript` to `currentUtteranceBuffer`.
        - Update `currentCard.appendFinal(transcript)`.
        - If `speech_final` is true (from metadata), call `finalizeCurrent()`.
    - Handle `UTTERANCE_END` message:
      - Call `finalizeCurrent()`.
    - `finalizeCurrent()` logic:
      - If `currentCard`, call `currentCard.finalize()`.
      - Clear `currentCard` reference and `currentUtteranceBuffer`.
  </action>
  <verify>
    <automated>grep -n "TRANSCRIPT_UPDATE" content.js && grep -n "UTTERANCE_END" content.js</automated>
  </verify>
  <done>The content script correctly drives the UI based on streaming events from the background.</done>
</task>
